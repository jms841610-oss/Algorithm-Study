투 포인터(Two Pointers)

\
\
\

개념 설명

투 포인터는 1차원 배열이나 리스트에서 두 개의 포인터(인덱스 위치를 가리키는 변수)를 조작하여 원하는 결과를 얻는 방법입니다. 

책상 위에 일렬로 나열된 숫자 카드를 양손의 검지 손가락으로 가리키면서 조건에 맞게 손가락을 요리조리 움직이는 모습을 상상하시면 딱 맞습니다.

\
\
\

작동 방식

투 포인터는 크게 두 가지 방식이 있는데, 가장 대표적인 **'양방향 포인터(서로를 향해 다가오는 방식)'**를 기준으로 설명해 드릴게요. 

(주로 정렬된 배열에서 두 수의 합을 찾을 때 씁니다.)

**왼쪽 포인터(Left)**는 배열의 맨 앞을, **오른쪽 포인터(Right)**는 배열의 맨 끝을 가리키게 합니다.

두 포인터가 가리키는 값을 더해봅니다.

합이 목표값보다 작다면: 값을 더 크게 만들어야 하므로, 더 큰 숫자가 있는 오른쪽으로 왼쪽 포인터를 한 칸 이동시킵니다.

합이 목표값보다 크다면: 값을 더 작게 만들어야 하므로, 더 작은 숫자가 있는 왼쪽으로 오른쪽 포인터를 한 칸 이동시킵니다.

합이 목표값과 같다면: 정답을 찾은 것이므로 탐색을 종료합니다! 두 포인터가 만나거나 교차할 때까지 이 과정을 반복합니다.

\
\
\

가정 및 제한사항

가정: 

양쪽 끝에서 시작하는 방식을 사용하려면, 이진 탐색 때와 마찬가지로 배열이 반드시 **오름차순으로 정렬(Sorted)**되어 있어야 한다는 가정이 필요합니다.

제한사항: 

투 포인터는 연속된 데이터(배열, 리스트)에서만 사용할 수 있습니다. 

또한, 구해야 하는 문제의 조건에 따라 포인터의 이동 방향(같은 방향으로 이동할지, 반대 방향으로 이동할지)을 잘 설계해야 합니다.

\
\
\

알고리즘 분석 (성능 및 대안)

시간 복잡도: 

두 포인터 각각이 배열을 최대 한 번씩만 훑으므로 O(N)입니다. 

중첩 반복문(O(N^2))으로 풀어야 할 문제를 선형 시간에 해결할 수 있게 해줍니다.

공간 복잡도: 

기존 배열 외에 포인터 변수 두 개 정도만 추가로 사용하므로 O(1)의 매우 효율적인 메모리 사용량을 보입니다.

대안 알고리즘:

누적 합(Prefix Sum): 구간의 합을 반복적으로 구해야 하고 데이터가 정적인 경우, 투 포인터보다 누적 합 배열을 미리 만드는 것이 유리할 수 있습니다.

슬라이딩 윈도우(Sliding Window): 구간의 길이가 고정되어 있다면, 투 포인터의 특수 형태인 슬라이딩 윈도우를 사용하여 더 단순하게 구현할 수 있습니다.

이진 탐색(Binary Search): 한 점을 고정하고 나머지 한 점을 찾을 때, 정렬된 상태라면 이진 탐색(O(N log N))을 고려할 수 있지만, 

투 포인터(O(N))가 가능하다면 투 포인터가 더 빠릅니다.
