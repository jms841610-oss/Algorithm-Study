다익스트라(Dijkstra)

\
\
\

개념 설명

다익스트라 알고리즘은 하나의 시작 노드에서 다른 모든 노드까지 가는 가장 짧은 거리(최단 경로)를 구하는 알고리즘입니다. 

이전의 BFS가 모든 간선의 길이가 같을 때 쓰였다면, 다익스트라는 간선마다 이동 비용(가중치)이 다를 때 사용합니다. 

고속도로 통행료나 도로별 소요 시간이 다를 때 가장 빠른 길을 찾는 것과 같습니다.

\
\
\

작동 방식

이 알고리즘은 비용이 가장 적게 드는 노드를 먼저 선택하기 위해 **우선순위 큐(Priority Queue)**를 핵심으로 사용합니다.

시작 노드의 거리는 0으로, 나머지 모든 노드까지의 거리는 무한대(아주 큰 값)로 초기화합니다.

우선순위 큐에서 현재 가장 거리가 짧은 노드를 꺼냅니다.

꺼낸 노드를 거쳐서 인접한 노드로 가는 비용을 계산해 봅니다.

만약 계산한 비용이 기존에 알고 있던 인접 노드까지의 거리보다 짧다면,

더 빠른 길을 찾은 것이므로 최단 거리를 갱신하고 우선순위 큐에 새 정보를 넣습니다.

큐가 빌 때까지 반복하면 모든 노드까지의 최단 거리가 계산됩니다.

\
\
\

가정 및 제한사항

가정: 

노드 간의 연결 정보와 비용을 담기 위해, pair를 활용한 인접 리스트와 효율적인 탐색을 위한 **우선순위 큐(Min-Heap)**를 사용한다고 가정합니다.

제한사항: 

다익스트라 알고리즘은 음수(-) 가중치를 가진 간선이 있는 경우에는 사용할 수 없습니다. 

거꾸로 갈수록 거리가 줄어드는 '타임머신' 같은 도로가 있다면 무한 루프에 빠지거나 잘못된 결과를 내기 때문입니다.
