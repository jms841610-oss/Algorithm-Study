DFS(깊이 우선 탐색)

\
\
\

개념 설명

DFS는 시작 노드에서 출발해서 다음 분기(branch)로 넘어가기 전에, 한 방향으로 갈 수 있을 때까지 아주 깊게 파고들며 탐색하는 방법입니다. 

미로 찾기에서 한쪽 길로 끝까지 가다가 막다른 길을 만나면, 마지막 갈림길로 되돌아와서 다른 길을 탐색하는 방식과 똑같습니다.

\
\
\

작동 방식

이 알고리즘은 함수가 자기 자신을 다시 부르는 재귀 호출(Recursion)을 사용하거나 스택(Stack) 자료구조를 사용하여 구현합니다. 

가장 직관적이고 널리 쓰이는 방법은 재귀 호출입니다.

현재 노드를 방문했다고 표시합니다.

현재 노드와 연결된 인접 노드 중, 아직 방문하지 않은 노드가 있다면 그 노드로 이동합니다.

이동한 새로운 노드에서 다시 동일한 탐색 과정(재귀)을 시작합니다.

더 이상 갈 곳이 없으면 이전 노드로 되돌아옵니다(백트래킹).

\
\
\

복잡도 분석

시간 복잡도: 

노드의 수 V, 간선의 수 E일 때, 모든 노드와 간선을 확인하므로 O(V + E)입니다.  (인접 리스트 기준)

공간 복잡도: 

최악의 경우 트리 모양 그래프에서 높이만큼의 스택 공간이 필요하므로 O(V)입니다. 

BFS에 비해 일반적으로 메모리 소모가 적은 편(너비가 넓고 깊이가 얕을 때)입니다.
