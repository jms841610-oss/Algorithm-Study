DFS(깊이 우선 탐색)

\
\
\

개념 설명

DFS는 시작 노드에서 출발해서 다음 분기(branch)로 넘어가기 전에, 

한 방향으로 갈 수 있을 때까지 아주 깊게 파고들며 탐색하는 방법입니다. 

미로 찾기에서 한쪽 길로 끝까지 가다가 막다른 길을 만나면, 

마지막 갈림길로 되돌아와서 다른 길을 탐색하는 방식과 똑같습니다.

\
\
\

작동 방식

이 알고리즘은 함수가 자기 자신을 다시 부르는 **재귀 호출(Recursion)**을 사용하거나 스택(Stack) 자료구조를 사용하여 구현합니다. 

가장 직관적이고 널리 쓰이는 방법은 재귀 호출입니다.

현재 노드를 방문했다고 표시합니다.

현재 노드와 연결된 인접 노드 중, 아직 방문하지 않은 노드가 있다면 그 노드로 이동합니다.

이동한 새로운 노드에서 다시 동일한 탐색 과정(재귀)을 시작합니다.

더 이상 갈 곳이 없으면 이전 노드로 되돌아옵니다(백트래킹).

\
\
\

가정 및 제한사항

가정: 

BFS 때와 마찬가지로, 그래프는 메모리 효율이 좋은 인접 리스트(Adjacency List) 방식으로 구현한다고 가정합니다.

제한사항: 

깊이를 우선으로 탐색하기 때문에, 찾은 경로가 항상 '최단 경로'라는 보장은 없습니다. 

또한 그래프의 깊이가 너무 깊을 경우 재귀 호출이 과도하게 쌓여 스택 오버플로우(Stack Overflow) 에러가 발생할 수 있다는 점을 주의해야 합니다.
