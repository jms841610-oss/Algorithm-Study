Union-Find

\
\
\

개념 설명

Union-Find는 여러 개의 노드가 존재할 때, 

두 개의 노드를 선택해서 현재 이 두 노드가 서로 같은 그래프(집합)에 속해 있는지를 판별하는 알고리즘입니다.

서로 중복되지 않는 부분 집합들로 나누어진 원소들을 관리하기 때문에 'Disjoint Set(서로소 집합)'이라고도 불립니다. 

마치 여러 개의 작은 파티션으로 나뉜 공간들을 필요에 따라 합치거나, 특정 인원이 어느 방에 있는지 확인하는 과정과 비슷합니다.

\
\
\

작동 방식

이 알고리즘은 각 노드의 '부모'가 누구인지를 저장하는 배열을 핵심으로 사용하며, 

크게 두 가지 연산으로 구성됩니다.

Find (찾기): 특정 노드가 속한 집합의 '대표 노드(루트)'를 찾는 과정입니다. 재귀적으로 부모를 타고 올라가 최상위 노드를 확인합니다.

Union (합치기): 두 노드가 속한 각 집합을 하나로 합치는 과정입니다. 보통 한쪽 루트 노드의 부모를 다른 쪽 루트 노드로 설정하여 연결합니다.

반복: 모든 연산은 배열의 인덱스를 참조하여 이루어지며, 합치기 요청이 들어올 때마다 트리 구조가 갱신됩니다.

\
\
\

가정 및 제한사항

가정

배열 기반 구현: 

각 노드의 부모 정보를 저장하기 위해 O(n) 크기의 1차원 배열을 사용한다고 가정합니다.

초기 상태에서 모든 노드의 부모는 자기 자신입니다.

최적화 적용: 

성능 극대화를 위해 Path Compression(경로 압축)을 사용하여 Find 연산 시 만나는 모든 노드가 직접 루트를 가리키도록 한다고 가정합니다.

제한사항

간선 제거의 어려움: 

Union-Find는 집합을 '합치는' 연산에는 매우 효율적이지만, 

이미 합쳐진 연결을 다시 '끊는(Disconnect)' 연산을 구현하기는 매우 까다롭고 비용이 많이 듭니다.

정적인 구조: 

주로 노드의 개수가 고정된 상태에서 연결성(Connectivity)을 다룰 때 유리하며, 

그래프의 형태가 실시간으로 복잡하게 변하는 모든 상황의 대안이 될 수는 없습니다.

\
\
\

알고리즘 분석 (성능 및 대안)

시간 복잡도: 

최적화(경로 압축 및 Union by Rank)를 적용할 경우, 거의 상수 시간에 가까운 O(alpha(n))을 보장합니다. 

이는 사실상 O(1)과 다름없을 만큼 빠릅니다.

공간 복잡도: 

각 노드의 부모(및 랭크)를 저장해야 하므로 O(n)의 메모리가 필요합니다.

대안 알고리즘: 

단순히 두 노드가 연결되었는지 한 번만 확인한다면 BFS나 DFS를 쓸 수 있습니다. 

하지만 실시간으로 간선이 추가되는 환경에서 반복적으로 연결을 확인해야 한다면 Union-Find가 훨씬 효율적인 선택지입니다.
