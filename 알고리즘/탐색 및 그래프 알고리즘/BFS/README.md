BFS(너비 우선 탐색)

\
\
\

개념 설명

BFS는 그래프나 트리 구조에서 시작 노드에 인접한 노드부터 먼저 탐색하는 방법입니다. 

즉, 깊게 파고들기 전에 넓게 먼저 탐색하는 방식이라고 생각하시면 됩니다. 

물결이 퍼져나가는 모습을 상상하시면 이해하기 쉽습니다.

\
\
\

작동 방식

이 알고리즘은 먼저 들어온 데이터를 먼저 처리하는 큐(Queue) 자료구조를 핵심으로 사용합니다.

시작 노드를 큐에 넣고 '방문했음'으로 표시합니다.

큐에서 노드를 하나 꺼냅니다.

꺼낸 노드와 연결되어 있으면서 아직 방문하지 않은 모든 인접 노드를 큐에 넣고 '방문했음'으로 표시합니다.

큐가 완전히 빌 때까지 이 과정을 반복합니다.

\
\
\

가정 및 제한사항

가정: 

여기서 보여드릴 코드는 메모리를 효율적으로 사용하기 위해 그래프를 인접 리스트(Adjacency List) 방식으로 구현한다고 가정합니다.

제한사항: 

BFS는 모든 간선의 비용이 동일할 때 '최단 경로'를 찾는 데 매우 유용합니다. 

하지만 깊이 우선 탐색(DFS)에 비해 큐에 많은 노드를 저장해야 할 수 있어서, 

그래프의 규모가 매우 크면 메모리 사용량이 많아진다는 제한사항이 있습니다.
