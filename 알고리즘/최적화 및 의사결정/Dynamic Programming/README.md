동적 계획법(Dynamic Programming)

\
\
\

개념 설명

동적 계획법은 아주 거대하고 복잡한 문제를 작고 쉬운 문제들로 쪼개서 푼 다음, 

그 결과를 '기억'해 두었다가 나중에 다시 써먹는 방식입니다. 

한 번 푼 문제는 메모장에 적어두고, 

똑같은 문제가 다시 나오면 계산하지 않고 메모장에서 정답만 쏙 빼서 쓰는 '기억하며 풀기'라고 생각하시면 아주 쉽습니다.

\
\
\

작동 방식

DP를 구현하는 방법에는 크게 두 가지가 있는데, 

여기서는 가장 기본적이고 직관적인 바텀업(Bottom-up, 반복문 사용) 방식을 알려드릴게요. 피보나치 수열을 예로 들어보겠습니다.

작은 문제들의 결괏값을 저장해 둘 배열(DP 테이블)을 만듭니다.

가장 작은 문제들의 정답(초기값)을 배열에 미리 적어둡니다. (예: 0번째 피보나치 수는 0, 1번째는 1)

작은 문제들의 정답을 조합해서 바로 다음 단계의 정답을 구하는 규칙인 점화식(Recurrence Relation)을 찾습니다.

반복문을 돌면서 점화식에 따라 DP 테이블을 처음부터 끝까지 차례대로 채워나갑니다.

배열의 마지막에 도달하면 우리가 원하던 최종 정답이 완성됩니다!

\
\
\

복잡도 분석

시간 복잡도: 

중복 계산을 피하기 때문에, 보통 (상태의 개수 X 각 상태를 처리하는 시간)으로 결정됩니다. 

예를 들어, 피보나치 수열을 단순 재귀로 풀면 O(2^n)이지만, DP를 쓰면 O(n)으로 획기적으로 줄어듭니다.

공간 복잡도: 

결과를 저장할 배열 공간인 O(n) 혹은 O(n^2)이 필요합니다.
