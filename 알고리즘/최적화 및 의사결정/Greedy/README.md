그리디(Greedy) 

\
\
\

개념 설명

그리디 알고리즘은 이름 그대로 '현재 상황에서 당장 가장 좋아 보이는 것'만 탐욕스럽게 선택하는 방법입니다. 

나중의 일은 복잡하게 생각하지 않고, 매 순간 가장 최선이라고 생각되는 결정을 내리는 아주 직관적인 방식이죠.

\
\
\

작동 방식

일상생활의 **'거스름돈 문제'**를 생각하면 원리를 가장 쉽게 이해할 수 있습니다. 

카운터 직원으로서 손님에게 1260원을 거슬러 줘야 할 때, 동전을 가장 적게 주려면 어떻게 할까요?

가장 큰 단위의 동전(500원)부터 무조건 줄 수 있는 만큼 최대한 많이 거슬러 줍니다.

더 이상 500원짜리를 줄 수 없게 되면, 그다음으로 큰 단위(100원)로 넘어갑니다.

이런 식으로 남은 금액이 0이 될 때까지 동전 단위를 점차 줄여가며 선택을 반복합니다.

\
\
\

가정 및 제한사항

가정: 

매 순간 내린 최적의 선택이 모여 전체 문제의 최적의 해답이 되어야 합니다. 

거스름돈 문제에서는 큰 단위의 동전이 항상 작은 단위 동전의 '배수' 형태여야만 이 가정이 성립합니다.

제한사항: 

현재의 선택이 나중의 선택에 미치는 영향을 고려하지 않으므로, 항상 100% 최적의 결과를 보장하지는 않습니다. 

예를 들어, 눈앞의 마시멜로 1개를 지금 당장 먹는 것보다 참고 나중에 2개를 먹는 것이 더 이득인 상황에서는 그리디 알고리즘이 실패합니다.
