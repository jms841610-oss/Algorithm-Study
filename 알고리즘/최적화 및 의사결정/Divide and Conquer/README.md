분할 정복(Divide and Conquer)

\
\
\

개념 설명

분할 정복은 말 그대로 '나누고(Divide)' 그리고 **'정복(Conquer)'**하는 알고리즘입니다. 

두꺼운 전화번호부 책을 반으로 찢고, 또 반으로 찢어서 얇게 만든 뒤에 이름을 찾는 과정을 상상해 보세요. 

덩치가 큰 문제를 아주 작은 단위의 문제로 잘게 나눈 뒤, 각각을 해결하고 그 결과들을 다시 모아서 원래 문제의 답을 내는 방식입니다.

\
\
\

작동 방식

이 알고리즘은 보통 **재귀 함수(Recursion)**를 사용하여 다음 3가지 단계를 거쳐 작동합니다.

분할 (Divide): 현재 직면한 큰 문제를 같은 성격을 가진 더 작은 부분 문제들로 쪼갭니다. (보통 반으로 나눕니다.)

정복 (Conquer): 더 이상 쪼갤 수 없을 만큼 문제가 작아지면, 그 작은 문제들을 풉니다.

결합 (Combine): 정복해서 얻어낸 작은 문제들의 해답을 차곡차곡 합쳐서 전체 문제의 해답을 만들어냅니다.

\
\
\

가정 및 제한사항

가정: 

쪼개진 작은 문제들은 원래의 큰 문제와 완전히 똑같은 성격(구조)을 가지고 있어야만 재귀적으로 해결할 수 있습니다. 

또한, 앞서 배운 DP와 달리 분할 정복은 작은 문제들이 서로 중복되지 않는 경우(예: 배열의 왼쪽 절반과 오른쪽 절반)에 주로 사용합니다.

제한사항: 

문제를 쪼개기 위해 재귀 함수를 아주 깊게 호출해야 하므로, 함수 호출에 따른 메모리(스택) 사용량이 많아집니다.
