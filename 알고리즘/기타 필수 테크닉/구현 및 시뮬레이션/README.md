구현 및 시뮬레이션

\
\
\

개념 설명

시뮬레이션은 현실 세계의 규칙이나 게임의 로직을 컴퓨터 환경에서 똑같이 돌아가도록 흉내 내는 것입니다. 

엘리베이터의 작동 방식, 로봇 청소기의 움직임, 혹은 테트리스 블록이 떨어지는 규칙 등을 코드로 옮기는 모든 작업이 여기에 속합니다.

\
\
\

작동 방식 

(방향 벡터 활용)가장 많이 출제되고 실무에서도 자주 쓰이는 **'2차원 배열(지도) 위에서의 캐릭터 이동'**을 예로 들어 작동 방식을 설명해 드릴게요.

동, 서, 남, 북으로 이동할 때 $x$좌표와 $y$좌표가 어떻게 변하는지를 미리 배열(dx, dy)로 선언해 둡니다. 

(이를 방향 벡터라고 부릅니다.)캐릭터에게 이동 명령이 주어지면, 현재 위치에서 방향 벡터의 값을 더해 **'다음 위치'**를 미리 계산해 봅니다.

계산된 다음 위치가 지도의 범위를 벗어나지 않는지(예외 처리) 꼼꼼하게 확인합니다.

범위를 벗어나지 않는 안전한 이동이라면, 캐릭터의 현재 위치를 방금 계산한 다음 위치로 확실하게 갱신합니다.

\
\
\

가정 및 제한사항

가정: 

이 유형은 컴퓨터의 연산 속도(시간 복잡도)를 아슬아슬하게 줄이는 것보다는, 

코드가 길어지더라도 요구사항을 누락 없이 정확하게(예외 처리) 구현하는 것이 훨씬 중요하다고 가정합니다.

제한사항: 

조건이 많고 복잡할수록 코드의 길이가 길어지고 버그(오타, 범위 이탈 등)가 발생하기 쉽습니다. 

따라서 변수 이름을 알기 쉽게 짓고 주석을 꼼꼼히 다는 습관이 필수적입니다.
