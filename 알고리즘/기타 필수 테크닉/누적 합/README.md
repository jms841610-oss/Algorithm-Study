누적 합(Prefix Sum)

\
\
\

개념 설명

누적 합은 배열의 처음부터 특정 위치까지의 원소들을 미리 다 더해서 '새로운 배열'에 메모해 두는 기법입니다.

은행 계좌의 '잔고'를 생각하면 아주 쉽습니다. 

매번 거래 내역을 처음부터 끝까지 더해서 현재 잔고를 계산하지 않죠? 

거래가 일어날 때마다 이전 잔고에 더하거나 빼서 기록해 두면, 언제든 현재 잔고를 1초 만에 알 수 있는 것과 같은 원리입니다.

\
\
\

작동 방식

이 알고리즘은 '미리 계산해 두기(전처리)'와 '구간 합 구하기' 두 단계로 나뉩니다.

미리 계산해 두기: 

원본 배열과 크기가 같은(혹은 계산을 편하게 하기 위해 1칸 더 큰) 누적 합 배열을 만듭니다. 

첫 번째 원소부터 차례대로 이전 누적 합에 현재 원소를 더해서 기록합니다.

구간 합 구하기: 

예를 들어 3번째부터 7번째 원소까지의 합을 구하고 싶다면, 반복문으로 3번부터 7번까지 더할 필요가 없습니다. 

'7번째까지의 누적 합'에서 '2번째까지의 누적 합'을 빼기만 하면 순식간에 정답이 나옵니다!

\
\
\

복잡도 분석

시간 복잡도: 

* 전처리: 배열을 한 번 훑어야 하므로 O(N)입니다.

* 조회(Query): 배열의 값 두 개만 빼면 되므로 O(1)입니다.

공간 복잡도: 

합 정보를 저장할 O(N)의 추가 공간이 필요합니다.
