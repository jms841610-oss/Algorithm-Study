std::unordered_map의 핵심 동작 원리 개요

std::unordered_map은 이름 그대로 순서(정렬)가 없는 맵입니다. 

이를 구현하기 위해 이진 탐색 트리 대신 해시 테이블과 **배열(버킷)**을 사용합니다.

\
\
\

핵심 원리는 다음과 같습니다.

해시 함수(Hash Function): 

입력된 키(Key)를 수학적으로 계산하여, 

배열의 특정 인덱스(방 번호)로 즉시 변환합니다.

\
\
\

버킷(Bucket): 

해시 함수가 알려준 인덱스로 단번에 찾아가 데이터를 넣거나 뺍니다. 

탐색 과정 없이 바로 주소를 찾아가므로 속도가 예술입니다 ($O(1)$).

\
\
\

충돌 해결 (Separate Chaining): 

만약 서로 다른 두 개의 키가 해시 함수를 거쳤는데 우연히 '같은 방 번호'를 배정받으면 어떻게 될까요? 

이를 '해시 충돌(Collision)'이라고 합니다. 

C++ 표준 라이브러리는 주로 이 방에 **연결 리스트(Linked List)**를 매달아서 여러 데이터를 줄줄이 엮어두는 방식으로 이 문제를 해결합니다.
