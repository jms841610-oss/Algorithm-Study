std::deque의 동작 원리 

std::vector가 하나의 거대한 연속된 메모리 덩어리라면, std::deque는 일정한 크기의 메모리 블록(Chunk)들이 여러 개 연결된 형태입니다.

std::deque를 구현하기 위해 관리해야 할 핵심 요소는 다음과 같습니다.

Map (포인터 배열): 

데이터를 직접 저장하지 않고, 실제 데이터가 들어있는 '블록(Chunk)'들의 주소를 저장하는 관리 배열입니다.

Chunk (데이터 블록): 

고정된 크기(예: 요소 4개, 64개 등)를 갖는 배열들입니다. 데이터는 이곳에 저장됩니다.

Start Index & Size: 

데이터가 양방향으로 늘어날 수 있기 때문에, 

첫 번째 데이터가 전체 논리적 공간의 어디에 위치하는지(start_idx), 그리고 데이터가 총 몇 개인지(size)를 추적합니다.

데이터가 앞(push_front)이나 뒤(push_back)로 추가되다가 블록이 꽉 차면, 

새로운 블록을 하나 할당해서 Map에 연결해 줍니다. 

만약 Map 자체의 공간이 부족해지면, Map만 더 크게 재할당하고 블록들은 중앙으로 재배치합니다.
