std::sort의 핵심 동작 원리 (IntroSort)

C++ 표준 라이브러리의 std::sort는 주로 **인트로 정렬(Introsort, Introspective Sort)**이라는 알고리즘을 사용합니다. 

이는 세 가지 정렬 알고리즘이 팀을 이뤄 각자의 약점을 보완하는 환상적인 구조입니다.

\
\
\

주전 에이스 (Quick Sort): 기본적으로 가장 빠른 퀵 정렬로 데이터를 분할하며 정렬을 시작합니다.

\
\
\

안전망 (Heap Sort): 퀵 정렬은 운이 나쁘면 시간이 엄청 오래 걸리는 치명적인 단점(최악의 경우 시간 복잡도 O(N^2))이 있습니다. 

그래서 분할이 너무 깊어지면(재귀 호출이 한계치에 다다르면) 퀵 정렬을 멈추고, 

최악의 경우에도 무조건 $O(N log N)$의 속도를 보장하는 힙 정렬로 스위칭합니다.

\
\
\

마무리 투수 (Insertion Sort): 데이터가 잘게 쪼개져서 배열의 크기가 아주 작아지면(보통 16개 이하), 

복잡한 정렬보다 단순 무식한 삽입 정렬이 훨씬 빠릅니다. 

이때는 삽입 정렬로 마무리를 짓습니다.
